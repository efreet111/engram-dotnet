# FlowDoc Work Cycle

> **Referencia opcional** — este documento describe el ciclo FlowDoc original. No es el proceso obligatorio de engram-dotnet; se incluye como referencia contextual.


## The 15-Day Cycle

FlowDoc uses 15-day cycles adapted for distributed teams:

| Day | Activity |
|-----|----------|
| 1-2 | Planning + Proposals |
| 3-11 | Implementation (parallel work with feature flags) |
| 12-14 | Integration + Review |
| 15 | Release + Retrospective |

## SDD Phases

Each change follows: **Explore → Proposal → Spec → Design → Tasks → Apply → Verify → Archive**

For details, see the full FlowDoc framework documentation.

## Status Conventions

- 🟡 In Progress — Active work
- 🟢 Done — Completed
- 🔴 Blocked — Waiting on something
- ⚫ Archived — Cancelled or superseded
